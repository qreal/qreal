Описание языков

В QReal:Databases можно создавать как логические, так и физические схемы БД. (см. раздел Создание диаграмм)

Логическая схема основана на модели "Сущность-связь" и содержит концептуальную информацию о БД - набор сущностей с атрибутами, связей между ними и ключевые свойства перечисленных элементов.

Физическая схема содержит информацию, специфичную для каждой СУБД - элементы "Таблица" и "Столбец" меняют набор измяемых свойств в зависимости от СУБД.

Ниже приведено описание элементов языков логической и физической схем, поддерживаемых средой моделирования QReal:Databases.

Язык логической схемы БД

 Entity (сущность)
Класс объектов, информацию о которых необходимо хранить.
Attribute (атрибут сущности)
Свойство сущности, которое хотим учитывать при построении модели. Атрибут располагается внутри элемента "сущность", которому принадлежит.
One-to-one relationship (связь один-к-одному)
Связь между двумя сущностями, которая означает, что одному объекту первой сущности соответствует один объект второй сущности.
One-to-many relationship (связь один-ко-многим)
Связь между двумя сущностями, которая означает, что одному объекту первой сущности может соответствовать несколько объектов второй сущности. Связь изображается в виде стрелки с двумя разными концами: два штриха - для первой сущности, разветвление - для второй сущности.
Many-to-many relationship (связь многие-ко-многим)
Связь между двумя сущностями, которая означает, что одному объекту первой сущности может соответствовать несколько

Язык физической схемы БД

 Table (таблица)
Элемент, который будет сгенерирован как таблица базы данных.
Column (столбец)
Атрибут таблицы, который будет сгенерирован как столбец таблицы БД, которой принадлежит. Располагается внутри элемента "Таблица".
One-to-many relationship (связь один-ко-многим)
Cвязь, предназначенная для соединения участников отношения один-ко-многим. Присутствует в схеме для наглядности.
Many-to-many relationship (связь многие-ко-многим)
Cвязь, предназначенная для соединения участников отношения многие-ко-многим со сгенерированной для отношения таблицей. Присутствует в схеме для наглядности.

название тип значения описание

В противном случае таблица также создана не будет, но команда вызовет ошибку. При создании сравниваются только имена таблиц.
Avg row length целое число Приближенное значение средней длины строки для данной таблицы. Имеет смысл устанавливать только для обширных таблиц с записями переменной длины.
Check sum {1 | 0} Следует установить в 1, чтобы в MySQL поддерживалась проверка контрольной суммы для всех строк (это делает таблицы немного более медленными при обновлении, но позволяет легче находить поврежденные таблицы).
Max rows целое число Максимальное число строк, которые планируется хранить в данной таблице.
Min rows целое число Минимальное число строк, которые планируется хранить в данной таблице.
Pack keys УДАЛИТЬ {0 | 1 | DEFAULT} Следует установить в 1 для получения меньшего индекса. Обычно это замедляет обновление и ускоряет чтение (MyISAM, ISAM). Установка в 0 отключит уплотнение ключей. При установке в DEFAULT (MySQL 4.0) обработчик таблиц будет уплотнять только длинные столбцы CHAR/VARCHAR.
Delay key write УДАЛИТЬ {0 | 1} Установка в 1 задерживает операции обновления таблицы ключей, пока не закроется указанная таблица (MyISAM).
Auto increment целое число Устанавливает для данной таблицы следующее значение AUTO_INCREMENT.
Type строка Задает тип создаваемой таблицы.

Возможные типы таблиц в MySQL:
- BDB - Таблицы с поддержкой транзакций и блокировкой страниц.
- HEAP - Данные этой таблицы храняться только в памяти.
- ISAM - Оригинальный обработчик таблиц.
- InnoDB - Таблицы с поддержкой транзакций и блокировкой строк.
- MERGE - Набор таблиц MyISAM, используемый как одна таблица.
- MRG_MYISAM - Псевдоним для MERGE.
- MyISAM - Новый обработчик, обеспечивающий переносимость таблиц в бинарном виде, который заменяет ISAM.

Primary key логическое Задает первичный ключ таблицы. В таблице может быть задан только один первичный ключ. Данный столбец должен быть уникальным и ненулевым.
Unique логическое Задает столбец, содержащий уникальные значения (значения не повторяются для разных строк).
Not null логическое Указывает, что данный столбец не может содержать значение NULL.
With compression логическое Включает сжатие столбца для оптимизации хранения. Параметр может быть использован со столбами типа CHARACTER, TEXT и их синонимами.
With comp логическое Включает сжатие столбца для оптимизации хранения. Параметр может быть использован со столбами типа CHARACTER, TEXT и их синонимами.
