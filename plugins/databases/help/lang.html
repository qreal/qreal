Описание языков

В QReal:Databases можно создавать как логические, так и физические схемы БД. (см. раздел Создание схемы БД)
Логическая схема основана на модели "Сущность-связь" и содержит концептуальную информацию о БД - набор сущностей с атрибутами, связей между ними и ключевые свойства сущностей (имя), атрибутов (имя, тип, является ли уникальным, является ли первичным ключом) и связей (имя).
Физическая схема содержит информацию, специфичную для каждой СУБД - элементы "Таблица" и "Столбец" меняют набор измяемых свойств в зависимости от СУБД. Ниже приведено описание элементов языков логической и физической схем, поддерживаемых средой моделирования QReal:Databases.

Язык логической схемы БД

 Entity (сущность)
Класс объектов, информацию о которых необходимо хранить. Свойство "name" должно быть непустым.
Attribute (атрибут сущности)
Свойство сущности, которое хотим учитывать при построении модели. Атрибут располагается внутри элемента "сущность", которому принадлежит. Свойство "name" должно быть непустым.
One-to-one relationship (связь один-к-одному)
Связь между двумя сущностями, которая означает, что одному объекту первой сущности соответствует один объект второй сущности.
One-to-many relationship (связь один-ко-многим)
Связь между двумя сущностями, которая означает, что одному объекту первой сущности может соответствовать несколько объектов второй сущности. Связь изображается в виде стрелки с двумя разными концами: два штриха - для первой сущности, разветвление - для второй сущности. Свойство "columnName" используется при генерации физической модели БД.
Many-to-many relationship (связь многие-ко-многим)
Связь между двумя сущностями, которая означает, что одному объекту первой сущности может соответствовать несколько объектов второй сущности и наоборот. Свойство "tableName" используется при генерации физической модели БД.


Язык физической схемы БД

 Table (таблица)
Элемент, который будет сгенерирован как таблица базы данных.
Column (столбец)
Атрибут таблицы, который будет сгенерирован как столбец таблицы БД, которой принадлежит. Располагается внутри элемента "Таблица".
One-to-many relationship (связь один-ко-многим)
Cвязь, предназначенная для соединения участников отношения один-ко-многим. Присутствует в схеме для наглядности.
Many-to-many relationship (связь многие-ко-многим)
Cвязь, предназначенная для соединения участников отношения многие-ко-многим со сгенерированной для отношения таблицей. Присутствует в схеме для наглядности.
